# 价格行为标注图质量闸门

## 必查项

交付前逐项检查：

- 是否说明了使用的事实源：用户图表、OHLC、交易记录、实时数据或条件性假设。
- 是否产出了一张可打开的标注分析图，或明确说明不能产图的原因。
- 标注图是否保留原 K 线图作为背景，没有重画、篡改或遮盖价格结构。
- 是否同时包含历史复盘层和当前计划层。
- 历史复盘层是否还原当时证据、当时动作/观望、当时失效点和后续验证，而不是只用后见之明命名。
- 当前计划层是否站在最后一根可见 K 线做下一步决策。
- 是否先判断市场周期，再评价形态质量。
- 是否给出持续方向偏向，或明确说明当前是区间中性/方向不明。
- 是否在图上标出关键位置，而不是只在正文描述涨跌。
- 是否同时给出多头和空头场景。
- 是否给出清楚的条件链：触发 -> 交易动作 -> 失效/止损 -> 目标 -> 不交易条件。
- 是否说明哪种证据会让当前判断失效。
- 是否包含不交易条件。
- 是否避免孤立 K 线、孤立形态和确定性预测。
- 图上文字是否足够短，不遮住最后一段走势、价格轴和关键 K 线。
- 图上文字是否全部留在图片边界内，没有右侧/底部截断。
- 图上文字是否有可读底板，并绘制在色块之上，没有被通道、区间或风险色块挡住。
- 图上主标签是否使用中文术语；除非用户要求，不用英文术语做主标签。

## 图表分析最低标准

必须包含：

```text
分析图：
历史复盘要点：
当前决策点：
证据链：
多头触发：
空头触发：
失效位置：
目标区域：
不交易条件：
```

如果用户提供的信息不足，改为：

```text
基于现有信息能判断：
无法判断：
继续分析需要：
条件性场景：
```

## 交易计划最低标准

必须包含：

```text
分析图：
当前是否可交易：
交易前提：
入场触发：
止损/失效：
目标区域：
不交易条件：
复盘观察点：
```

禁止只输出“做多/做空”。无法定义止损或失效点时，结论应降级为观察计划。

## 复盘最低标准

必须包含：

```text
分析图：
当时背景：
形态质量：
执行问题：
更优处理：
可复用规则：
```

复盘优先评价过程质量，不用事后结果倒推交易一定对错。

## 失败信号

出现以下情况时，先修正再交付：

- 只讲形态名字，没有解释背景。
- 只给一个方向，没有替代场景。
- 用“明显”“肯定”“必然”等词替代证据。
- 只有历史复盘，没有当前计划。
- 只有当前计划，没有分析复盘前面的关键交易点。
- 从左到右解释历史走势，但每个历史点只说“发生了什么”，没有说明当时证据、当时计划和当时失效点。
- 把历史最高点、最低点、高潮、反转都贴上名字，却没有说明当时能不能交易。
- 有当前触发位、失效位和不交易条件，但完全没有复盘前文关键结构。
- 只有结论，没有 `因为...所以...但只有...才...如果...失效` 的证据链。
- 只有文字分析，没有在用户图上生成标注产物。
- 图上标签堆叠成大段文章，影响读图。
- 图上标签超出图片边界、被裁切，或贴住价格轴导致读不完整。
- 图上标签被半透明色块压住，或没有底板导致在深色/复杂 K 线区域不可读。
- 图上大量使用英文术语，用户需要二次翻译才能看懂。
- 把区间中央的随机波动讲成高质量形态。
- 忽略突破后的后续跟随。
- 没有说明交易失败后在哪里认错。
- 在没有实时数据或图表时，给出具体价格操作。
