# 标注图规格

## 目标

把用户给出的 K 线截图做成交易教学分析图：保留原图作为背景，在同一张图上叠加历史复盘标注和接下来交易计划标注。图后再输出复盘要点和交易建议。

不要使用生成式图片工具重画 K 线图。必须优先在原图上做确定性叠加，避免篡改价格、K 线和坐标轴。

默认产物不是纯历史讲解图，也不是纯盘前计划图，而是“复盘层 + 计划层”的合成图。复盘层解释前面关键机会的当时逻辑，计划层回答最后可见价格附近接下来怎么做。

## 标注风格

- 多头、反弹、买入机会：绿色或青绿色。
- 空头、下跌、卖出机会：红色或玫红色。
- 震荡区间、窄幅震荡、等待突破：橙色或黄色半透明。
- 弱趋势通道、下降通道、风险区域：红色半透明。
- 测量目标、关键价位：黑色水平线或虚线。
- 文字标签：默认中文，短句，不超过 4 行；图上写结论，正文解释原因。
- 文字标签默认带半透明深色底板，最后绘制，压在色块之上，避免被通道/区间色块挡住。
- 图上保留足够空白，不遮住最后一段价格行为和价格轴；右侧标签必须留在图片内。
- 除非用户明确要求，图上不要写 `Always in`、`Bear Surprise`、`Sell Climax`、`TTR` 等英文词；改写为“持续多头/持续空头”“强空头反转”“卖压高潮”“窄幅震荡”。

## 标注元素优先级

同一张图必须包含两组元素。

历史复盘层优先级：

1. 关键转折点：强突破、失败突破、趋势转区间、区间边缘失败、高潮后反弹。
2. 当时证据：当时已经能看到的 K 线力量、位置、后续跟随、回调质量。
3. 当时计划：当时应该顺势、等二次进场、离场、观望，还是只做 scalp。
4. 当时失效：如果哪里被突破或收回，原判断失效。
5. 后续验证：后面走势如何验证或推翻当时计划。

当前计划层优先级：

1. 当前价格所在位置：区间中部、边缘、突破后、回测位、趋势尾端或方向不明。
2. 下一步触发条件：上破并有后续跟随、下破并有后续跟随、失败突破、二次进场、回测失败。
3. 失效/止损位置：多头错在哪里、空头错在哪里。
4. 目标区域：最近阻力/支撑、区间另一侧、测量目标。
5. 不交易条件：区间中部、距离失效点太远、风险回报不清、突破缺少跟随。

视觉权重规则：

- 历史复盘层占 50%-60%，当前计划层占 40%-50%；用户强调“接下来怎么做”时，当前计划层可更重。
- 历史复盘层不要贴满全图，只标关键交易点。宁可少标，也不要把每个波段都命名。
- 当前计划层必须存在。如果图上没有“现在不做/做多触发/做空触发/失效点”这类标签，说明标注失败。
- 历史复盘标签用偏淡底板，当前计划标签用更醒目的底板或颜色，避免两层混在一起。

## JSON 结构

`scripts/render_annotations.py` 接收如下 JSON：

```json
{
  "title": "可选标题",
  "annotations": [
    {
      "type": "zone",
      "points": [[0.12, 0.18], [0.43, 0.30], [0.42, 0.55], [0.10, 0.38]],
      "color": "#ef6f6c",
      "opacity": 0.16
    },
    {
      "type": "rect",
      "x": 0.73,
      "y": 0.47,
      "w": 0.20,
      "h": 0.17,
      "color": "#f2b84b",
      "opacity": 0.25
    },
    {
      "type": "line",
      "start": [0.15, 0.36],
      "end": [0.31, 0.43],
      "color": "#356dd8",
      "width": 3,
      "dash": "12 10"
    },
    {
      "type": "arrow",
      "start": [0.42, 0.20],
      "end": [0.45, 0.29],
      "color": "#168c7a",
      "width": 3
    },
    {
      "type": "text",
      "at": [0.60, 0.18],
      "text": "强空头反转\\n持续空头\\n等待二腿下跌",
      "color": "#c84f65",
      "size": 30,
      "max_width": 0.18,
      "box": true
    }
  ]
}
```

坐标优先使用 0 到 1 的归一化坐标，便于适配不同分辨率。脚本也接受像素坐标：当坐标值大于 1 时按像素处理。

## 元素类型

- `zone`：多边形区域，适合趋势通道、楔形、风险区域。
- `rect`：矩形区域，适合交易区间、窄幅震荡、横盘等待突破。
- `line`：直线或水平线，适合关键价位、测量目标、通道边界。
- `arrow`：方向箭头，适合买卖信号、趋势切换、突破失败。
- `text`：文字标签，适合写形态名称和交易含义。

## 文字标注规则

`text` 元素默认按左上角定位。渲染器会自动换行、夹住边界、添加底板，并把所有文字放到最上层。

可用字段：

- `text`：中文短标签。每行尽量不超过 12 个汉字，总行数不超过 4 行。
- `at`：文字框左上角坐标，优先用 0 到 1 的归一化坐标。
- `size`：字号。普通标签 28-34；密集图用 24-28。
- `max_width`：文字框最大宽度。建议 0.16-0.24；右侧标签不要超过 0.18。
- `box`：是否显示底板，默认 true。
- `box_fill`：底板颜色，默认 `#0b0f14`。
- `box_opacity`：底板透明度，默认 0.72。
- `padding`：底板内边距，默认 10。

避免：

- 把文字放在图片最右侧但不给 `max_width`。
- 把文字压在半透明区间内部且不加底板。
- 用英文术语堆标签。图上改成中文，正文可补充英文对应。

## 交易建议文本

图后必须给出：

```text
历史复盘要点：
当前决策点：
证据链：
多头触发：
空头触发：
失效位置：
目标区域：
不交易条件：
风险提醒：
```

`证据链` 必须按这个格式写，避免看图说话：

```text
因为 A（当前背景），所以 B（当前偏向/中性），但只有 C（触发证据）出现后，才执行 D（交易动作）；如果 E 发生，判断失效。
```

`历史复盘要点` 必须按这个格式写，避免事后诸葛亮：

```text
在 X 点，当时已经能看到 A（证据），所以合理计划是 B（动作/观望），失效点是 C；后面 D 说明这个计划被验证/推翻。
```

不合格：

```text
这里是上涨通道，那里是卖压高潮，后面反弹。
```

合格：

```text
在 20:00 冲高失败后，当时已经出现强反向阴线和缺少继续上攻，所以多头不应追高；若重新站回高点上方，空头反转判断失效。后面连续下跌验证了离场/等待空头信号的计划。

当前在反弹小区间中部，距离上下边界都不近，所以不追价；只有上破 59900 并有后续跟随，才等回测多；如果跌回 59500 下方且收不回，转为空头延续。
```

建议使用条件句，不使用确定性承诺：

```text
如果向上突破区间并有后续跟随，再考虑多头回测入场。
如果跌回区间内，向上突破应降级为失败突破。
如果仍在区间中央，等待，不追价。
```

不要只写：

```text
当前判断：
多头计划：
空头计划：
不交易条件：
```
