# 标注图规格

## 目标

把用户给出的 K 线截图做成专业价格行为训练标注图：保留原图作为背景，在同一张图上叠加逐根训练标注、历史复盘标注和接下来交易计划标注。图后再输出市场状态切换、关键 bar 复盘和从交易假设推导出来的交易建议。

不要使用生成式图片工具重画 K 线图。必须优先在原图上做确定性叠加，避免篡改价格、K 线和坐标轴。

默认产物不是纯历史讲解图，也不是纯盘前计划图，而是“训练层 + 复盘层 + 计划层”的合成图。训练层解释市场状态和关键 K 线如何一段段改变；复盘层解释前面关键机会的当时背景、证据和价格行为逻辑；计划层回答最后可见价格附近的交易假设、信号 K、入场 K、跟进 K、失败、止损和目标。

## 标注风格

- 多头、反弹、买入机会：绿色或青绿色。
- 空头、下跌、卖出机会：红色或玫红色。
- 震荡区间、窄幅震荡、等待突破：橙色或黄色半透明。
- 弱趋势通道、下降通道、风险区域：红色半透明。
- 测量目标、关键价位：黑色水平线或虚线。
- Bar 编号和推数：默认写进正文，不主动加到图上。只有原图已有 bar count，或用户明确要求逐根编号时，才少量补充。
- 信号 K、入场 K、跟进 K：用小箭头、短标签或连线标明三者关系；不要只把一条价格线当成入场理由。
- 测量运动：用水平线或括号式短线标出 A/B/C/D、区间高度、急速高度或 50% 回撤，并在正文说明锚点。
- 文字标签：默认中文，短句，不超过 5 行；必须写成“术语 + 人话解释 + 动作含义”。术语可以出现，但不能只贴术语。
- 文字标签默认带半透明深色底板，最后绘制，压在色块之上，避免被通道/区间色块挡住。
- 图上保留足够空白，不遮住最后一段价格行为和价格轴；右侧标签必须留在图片内。
- 除非用户明确要求，图上不要写 `Always in`、`Bear Surprise`、`Sell Climax`、`TTR` 等英文词；改写为“持续多头/持续空头”“强空头反转”“卖压高潮”“窄幅震荡”。
- 图例只在颜色或线型不直观时添加，最多两行。图例只解释读图方法，不写交易建议；如果图面已经拥挤，改在正文解释。

## 开放标注模式

标注前先问：这张图最需要用户看懂什么？然后按优先级选标注形式，不要把所有可用工具都画上去。

## 标注对象绑定

生成任何图上标注前，先做一张内部绑定表。没有通过绑定表的内容，不进入图上。

```text
图上对象：
对应哪段走势：
边界在哪里：
价格行为证据：
观众要学什么：
图上怎么标：
文字放哪里：
删除条件：
```

填写标准：

- `图上对象` 必须是看得见的结构，例如区间框、通道边界、水平线、失败突破、当前决策区域。
- `对应哪段走势` 必须能用时间位置或相邻结构描述，不能只写“这里”“右边”“中间”。
- `边界在哪里` 必须能落到图上的框、线或价格区域；说不清边界，就不要画色块或线。
- `价格行为证据` 必须写 K 线事实，例如突破没有跟进、强反向 K 打回、连续高低点下移、区间内重叠增加。
- `观众要学什么` 必须是一条可复用判断，例如“突破没有跟进时，不直接当成趋势成立”。
- `图上怎么标` 只能选一种主视觉：框、水平线、通道边界、计划面板或正文说明。
- `文字放哪里` 必须贴近对象；如果只能放很远，优先把解释移到正文。
- `删除条件` 写清什么时候这条标注不该上图，例如对象边界不清、文字无法贴近、只是在复述结论。

通过标准：

- 观众不看正文，也能知道每个文字框解释哪一个框、哪一条线或哪一段走势。
- 图上文字不是独立观点，而是某个图上对象的标签。
- 每个对象只教一个重点，不能一个框同时讲区间、突破、测量和当前计划。

## 复盘训练标注

复盘训练图要先帮助用户看懂市场周期如何切换，再帮助用户模拟当时能不能做决定。图上不要塞完整交易日讲解，完整模拟问题放正文。

图上保留：

- 市场周期段：突破、宽通道、窄通道、交易区间。
- 周期切换信号：连续强 K、强反向 K、突破后跟进、重叠明显增加。
- 关键交易对象：信号 K 所在区域、入场触发线、止损/失效边界、目标磁体。
- 当前决策对象：最后价格附近是否可交易，以及需要等什么证据。

正文必须补齐：

```text
当时看到什么：
当时能不能入场：
信号 K：
入场 K：
跟进 K：
止损/失效：
目标：
提前离场条件：
如果错过，是否有二次机会：
下次可复用规则：
```

图上文字写法：

- `连续三根强阴线收低 -> 空头突破有跟进 -> 多头先离场，空头只等回调或继续跟进`
- `突破后马上回到框内 -> 突破失败 -> 不追突破，回到区间思路`
- `K线变小且重叠变多 -> 趋势转成区间 -> 中间不交易，等边缘或强突破`

不合格：

- `这里做空`
- `强突破`
- `完美机会`
- `后面跌了，所以这里应该空`

常用标注形式：

- 通道边界线：用于窄通道、宽通道、抛物线推进、下降通道。默认用两条线，不用色带。
- 楔形/三角区域：用于三推、楔形底/顶、扩张三角。
- 区间框：用于窄幅区间、宽幅区间、TTR、等待突破。
- 编号：只在原图已有 bar count，或用户明确要求逐根编号时使用。
- 圆圈/椭圆：默认不用。只有用户明确要求圈出某根 K 线，或单个信号点不用圈会看不清时，才少量使用。
- 水平线/短横线：用于前高前低、突破点、测量目标、50% 回调。
- 括号/测量线：用于一倍/两倍测量、AB=CD、区间高度投影。
- 虚线：用于趋势线、楔形边、尚未确认的计划线、弱边界。
- 粗箭头/小箭头：用于方向切换、信号触发、离场/入场提示。
- 指向线：低优先级工具。默认不用；只有文字无法靠近对象，且能明确说明指向哪个具体对象时才用。
- 图面说明段落：用于解释交易方式、概率、为什么等待，不必每句话都塞进 K 线旁边。
- 右侧计划面板：用于当前交易计划、失败条件、目标和不交易条件。

选择顺序：

1. 先用框线画出市场状态：区间用矩形框，通道用两条边界线。
2. 再用横线、区间框、通道线标出会改变动作的边界。
3. 再用少量箭头或贴近对象的文字定位关键变化，不默认加编号、圈选或指向线。
4. 再用虚线、括号线说明最重要的目标和测量。
5. 最后只保留少量文字解释；完整逐根复盘放到正文。

标注重要性排序：

1. 必须保留：当前市场状态、关键区间/通道/突破点、最后价格附近的交易计划。
2. 优先保留：会改变多空判断的支撑阻力、突破回测位、失败条件、最近目标磁体。
3. 空间足够才保留：测量运动、50% 反应区、前面历史机会的复盘说明。
4. 默认移到正文：逐根 bar 复盘、推数细节、信号 K/入场 K/跟进 K 的完整解释、术语展开。
5. 默认删除：与当前计划无关的历史波段命名、重复方向箭头、原图没有编号时新增的一串数字或圈号。

对象数量限制：

- 默认一张图最多保留 3-5 个对象。
- 第一优先级是当前决策对象；第二优先级是解释当前决策所需的最近结构；第三优先级才是远处历史复盘。
- 如果对象超过 5 个，先拆成“结构复盘图”和“当前计划图”，或把远处历史复盘移到正文。

自由度边界：

- 可以为了教学清晰输出两张图：结构复盘图和当前计划图。
- 可以保留英文术语，但必须配中文解释。
- 可以写较长教学段落，但默认放在正文；只有图上有足够空白且不会遮挡 K 线时，才放在空白处或右侧面板。
- 可以少标历史，只要当前计划要求更强；也可以弱化当前计划，只要用户要求纯训练复盘。
- 可以用颜色、线型、粗细表达确定性：实线表示已发生，虚线表示条件计划，淡色表示低确定性。

清晰度边界：

- 色块必须让人一眼看出含义：区间用矩形框；通道默认用两条边界线；风险区域只能用简单矩形或规则窄带。
- 不用手绘多边形去“包住走势”。如果多边形边界不能解释为通道线、楔形边或区间边界，就删除色块，改用一条趋势线或文字说明。
- 图上每一条线都必须有用途：支撑、阻力、通道边界、突破点或测量。说不清用途的线直接删除。
- 指向线默认不用。需要使用时，先写清“文字指向谁”，再画短、细、灰色线；如果它看起来像趋势线、压力线或交易信号，就删除。
- 文字必须写成“看到什么 -> 说明什么 -> 所以怎么做”。如果一句话需要懂术语才看得懂，就改成人话。
- 文字必须贴近对象。文字框如果离对象很远，就不是图上标注，而是正文说明；把它移到正文。

## 标注元素优先级

同一张图必须包含三组元素。

逐根训练层优先级：

1. 市场状态切换：急速、窄幅通道、宽幅通道、交易区间、突破测试、反转尝试、高潮。
2. 关键 bar 编号：强突破 bar、失败突破 bar、信号 bar、入场 bar、跟进 bar、高潮 bar、最后决策 bar。
3. 持续方向切换：持续多头、区间中性、持续空头，以及改变方向的证据。
4. 推数和形态依据：二推、三推、楔形、双顶/双底、末端旗形、抛物线推进。
5. 测量运动和磁体：AB=CD、区间高度投影、急速高度、50% 回撤、EMA20 首次触碰或穿越。
6. 降级处理：趋势变通道、通道变区间、突破变失败突破、高潮后转观察。

术语解释优先级：

1. 图上第一次出现术语时，用一行普通话解释它对交易的意思。
2. 术语解释必须回答“看到这个以后该做什么”：顺势、等回调、离场、缩小目标、等待突破、删除计划。
3. 同一个术语重复出现时，后面只写术语和动作，不重复长解释。
4. 如果一个术语不能用普通话解释清楚，先不要写到图上，改在正文解释。

历史复盘层优先级：

1. 关键转折点：强突破、失败突破、趋势转区间、区间边缘失败、高潮后反弹。
2. 当时背景：当时是趋势、通道、区间、突破测试、回调，还是反转尝试。
3. 当时证据：当时已经能看到的 K 线力量、位置、后续跟随、回调质量。
4. 价格行为逻辑：这些证据为什么支持顺势、观望、失败突破、突破回测或区间交易。
5. 当时计划：当时应该顺势、等二次进场、离场、观望，还是只做 scalp。
6. 当时失效：如果哪里被突破或收回，原判断失效。
7. 后续验证：后面走势如何验证或推翻当时计划。

当前计划层优先级：

1. 当前价格所在位置：区间中部、边缘、突破后、回测位、趋势尾端或方向不明。
2. 交易假设：当前计划属于失败突破、突破回测、趋势回调、区间看衰突破、急赴磁体、通道反转，还是观察计划。
3. 下一步触发条件：需要哪根信号 K、入场 K 或后续跟进来证明假设开始成立。
4. 失败/止损位置：价格走到哪里说明假设错了，止损应放在该证据之外。
5. 目标区域：最近阻力/支撑、区间中轴/另一侧、测量目标、AB=CD 或趋势极值。
6. 删除条件：什么情况出现后，这套计划不再成立，应改判或观望。
7. 不交易条件：区间中部、距离失败点太远、目标太近、突破缺少跟随。

视觉权重规则：

- 专业训练图中训练层和历史复盘层占 60%-70%，当前计划层占 30%-40%；用户强调“接下来怎么做”时，当前计划层可更重。
- 历史复盘层不要贴满全图，只标关键交易点。宁可少标，也不要把每个波段都命名。
- 当前计划层必须存在。如果图上没有“交易假设/触发/失败或止损/目标/删除条件”这类标签，说明标注失败。
- 历史复盘标签用偏淡底板，当前计划标签用更醒目的底板或颜色，避免两层混在一起。
- 关键 bar 编号默认不放上图。若必须使用，数量必须少于文字标签，且只服务于正文逐根复盘；原图没有 bar count 时，不新增一串数字。

## JSON 结构

`scripts/render_annotations.py` 接收如下 JSON：

```json
{
  "title": "可选标题",
  "annotations": [
    {
      "type": "zone",
      "points": [[0.12, 0.18], [0.43, 0.30], [0.42, 0.55], [0.10, 0.38]],
      "color": "#ef6f6c",
      "opacity": 0.16
    },
    {
      "type": "rect",
      "x": 0.73,
      "y": 0.47,
      "w": 0.20,
      "h": 0.17,
      "color": "#f2b84b",
      "opacity": 0.25
    },
    {
      "type": "line",
      "start": [0.15, 0.36],
      "end": [0.31, 0.43],
      "color": "#356dd8",
      "width": 3,
      "dash": "12 10"
    },
    {
      "type": "arrow",
      "start": [0.42, 0.20],
      "end": [0.45, 0.29],
      "color": "#168c7a",
      "width": 3
    },
    {
      "type": "text",
      "at": [0.60, 0.18],
      "text": "强空头反转\\n持续空头\\n等待二腿下跌",
      "color": "#c84f65",
      "size": 30,
      "max_width": 0.18,
      "box": true
    }
  ]
}
```

坐标优先使用 0 到 1 的归一化坐标，便于适配不同分辨率。脚本也接受像素坐标：当坐标值大于 1 时按像素处理。

## 元素类型

- `zone`：多边形区域，只用于边界清楚的趋势通道、楔形或风险区域。点位必须形成可解释的形状；不能用扭曲色块包住走势。
- `rect`：矩形区域，适合交易区间、窄幅震荡、横盘等待突破。
- `line`：直线或水平线，适合关键价位、测量目标、通道边界、50% 回撤和区间高度投影。
- `polyline`：多段折线，适合手动画趋势线、抛物线近似、复杂回调路径、测量路径。
- `bracket`：括号测量线，适合一倍/两倍测量、区间高度、Measured Move。
- `arrow`：方向箭头，适合买卖信号、趋势切换、突破失败、信号 K 到入场 K 的关系。
- `circle`：圆圈，属于低优先级定位工具。只有用户要求圈选，或单个信号点不用圈会看不清时使用。
- `ellipse`：椭圆，属于低优先级定位工具。优先用区间框、通道线或贴近对象的文字替代。
- `marker`：带数字或短词的圆形标记，默认不用。只有原图已有 bar count，或用户明确要求逐根编号时使用。
- `text`：文字标签，适合写价格行为假设、交易含义、失败条件、目标、bar 编号和推数。

## 文字标注规则

`text` 元素默认按左上角定位。渲染器会自动换行、夹住边界、添加底板，并把所有文字放到最上层。

可用字段：

- `text`：中文短标签。每行尽量不超过 12 个汉字，总行数不超过 4 行。
- `at`：文字框左上角坐标，优先用 0 到 1 的归一化坐标。
- `size`：字号。普通标签 28-34；密集图用 24-28。
- `max_width`：文字框最大宽度。建议 0.16-0.24；右侧标签不要超过 0.18。
- `box`：是否显示底板，默认 true。
- `box_fill`：底板颜色，默认 `#0b0f14`。
- `box_opacity`：底板透明度，默认 0.72。
- `padding`：底板内边距，默认 10。
- `leader_start` / `leader_end`：可选，默认不用。只有文字无法靠近对象，且已确认读者不会把这条线误解成趋势线、压力线或交易信号时使用。
- `leader_color`：可选。指向线颜色默认灰色；不要使用红色、绿色这类容易被误读成交易信号的颜色。

关键 bar 编号使用条件：

- 先检查原图是否已有 bar count。已有时，只补解释，不重复覆盖原编号。
- 原图没有 bar count 时，默认不新增编号；用贴近结构的文字、短横线或正文复盘替代。
- 用户明确要求逐根编号时，才用 `text` 或少量 `marker` 标出 `信号`、`入场`、`跟进` 这类短词。
- 编号贴近 K 线的上方或下方，不放到价格轴、时间轴或实体正中。
- 如果图上已有软件 bar count，只补关键解释，不重复覆盖原编号。
- 如果截图分辨率低导致数字会糊，删除编号层，改在正文列“关键 bar 复盘”。

图上说明写法：

- 第一行写看到什么：`冲高后被大阴线打回`、`价格在框内来回走`、`右侧高点低点下移`。
- 第二行写说明什么：`高位追多容易被套`、`上下两边都没走成趋势`、`买方还没拿回主动`。
- 第三行写动作：`先防回调，不追多`、`中间不做，等靠边或突破`、`站回关键位前先观察`。
- 如果还需要失效条件，第四行写：`若重新站回高点，删除空头`。

例子：

```text
价格在框内来回走
上下都没走成趋势
中间先不做
```

不合格：

```text
B7 Sell Climax
```

避免：

- 把文字放在图片最右侧但不给 `max_width`。
- 把文字压在半透明区间内部且不加底板。
- 用英文术语堆标签。图上改成中文，正文可补充英文对应。

## 交易建议文本

图后必须给出：

```text
市场状态切换：
关键 bar 复盘：
持续方向：
推数与测量：
历史复盘要点：
当前判断：
交易假设：
证据链：
信号 K/入场 K/跟进 K：
失败/止损：
目标区域：
删除条件：
不交易条件：
风险提醒：
```

`证据链` 必须按这个格式写，避免看图说话：

```text
因为 A（当前背景），所以提出 B（交易假设），但只有 C（触发证据）出现后，才执行 D（交易动作）；如果 E 发生，说明假设失效，计划删除或降级为观察。
```

`历史复盘要点` 必须按这个格式写，避免事后诸葛亮：

```text
在 X 点，当时背景是 A，已经能看到 B（证据），这符合 C（价格行为逻辑），所以合理计划是 D（动作/观望），失效点是 E；后面 F 说明这个计划被验证/推翻。
```

`关键 bar 复盘` 必须按这个格式写，避免只贴术语：

```text
Bar X：当时市场状态是 A，这根 K 线提供 B 证据，所以持续方向/交易计划改成 C；如果下一根没有 D 跟进，就降级为 E。
```

不合格：

```text
这里是上涨通道，那里是卖压高潮，后面反弹。
上破做多，下破做空。
```

合格：

```text
在 20:00 冲高失败后，当时背景是区间上沿测试，已经出现强反向阴线和缺少继续上攻，这符合区间上沿失败突破逻辑，所以多头不应追高，空头可以等二次信号；若重新站回高点上方，空头假设失效。后面连续下跌验证了离场/等待空头信号的计划。

当前在反弹小区间中部，距离上下边界都不近，所以交易假设先是观察；只有上破 59900 并有后续跟随，才改成突破回测多头假设，止损放在回测失败点外，目标先看上一波高点；如果突破后跌回区间，删除多头计划。
```

建议使用条件句，不使用确定性承诺：

```text
如果向上突破区间并有后续跟随，交易假设才从观察改为突破回测多头。
如果跌回区间内，突破回测假设失效，删除多头计划。
如果仍在区间中央，等待，不追价。
```

不要只写：

```text
当前判断：
多头计划：
空头计划：
不交易条件：
```

如果同一张图需要同时列多头和空头，必须先说明它们对应不同的价格行为假设，且不会在同一个位置重复承担同一风险。

价格位不能单独作为触发。写 `站上 60650 做多` 或 `跌破 60350 做空` 前，必须改成 `突破 K 收在 60650 上方 -> 跟进 K 不跌回突破点 -> 回测形成多头信号 K -> 入场 K 突破信号 K 高点` 这种可复盘链条；说不清就只把价格位标为磁体或观察线。
