# 价格行为分析

## 核心原则

这个 skill 的主产物是一张教学型标注分析图，而不是单纯文字解读。用户给出 K 线图时，必须尽量在原图上叠加价格行为标注，并附上从图表证据推导出来的交易计划。

把用户提供的图表、OHLC 数据或明确行情片段作为唯一事实源。不要凭空补图、补价格、补成交细节；如果用户问当前行情、最新价格或具体标的的实时判断，先获取最新数据或要求用户提供图表，再进入分析。

使用阿布价格行为学作为主要分析语言：先判断背景和市场周期，再看具体形态。图上文字默认使用中文术语；除非用户要求保留英文，英文术语只放在正文补充，不放进主标注。

价格行为分析不是“上破做多、下破做空”的条件罗列。每个可交易计划都必须先说清交易假设，例如“区间下沿失败跌破后回到中轴”“空头趋势中的弱反弹二次做空”“强突破后的回测多头”“通道尾端反转尝试”。如果说不清属于哪一种价格行为假设，就降级为观察，不给入场计划。

交易计划必须按同一条逻辑链推导：市场周期 -> 当前价格位置 -> 交易假设 -> 入场触发 -> 失败条件 -> 止损位置 -> 目标磁体 -> 删除条件。止损来自“这个判断在哪里被证明错”，目标来自前高前低、区间中轴/边界、突破点、测量运动、AB=CD、趋势极值或其它图上磁体；不能先给方向再随手补止损和目标。

读图只抓两件事：脉络和动能。脉络回答市场现在处于趋势、通道、区间、突破还是测试；动能回答多空谁在推进、推进是否连续、是否有跟进。任何形态名称都必须服从这两项判断。

默认把市场理解为“动态寻找公平价”的过程。读图时先判断价格是在均衡区间内来回测试，还是已经进入寻找新均衡的失衡运动；突破先按边界测试处理，只有出现强突破和后续跟随，才升级为新一段趋势或新交易区间的探索。

把每一根 K 线都视为有交易理由的成交结果。逐根读取信息，但只把会改变交易计划的关键点画到图上；不要把看不懂的 K 线称为噪音，也不要用“庄家操纵”“割韭菜”替代可验证的多空证据。

图表事实优先于新闻、基本面叙事和多周期暗示。用户同时给出这些信息时，先回到当前图表、当前周期和最近 80-100 根 K 线；除非用户明确要求多周期分析，否则不要让更大周期或新闻理由覆盖当前图表上的止损和失效条件。

输出目标是“一张图同时完成教学型历史复盘和下一步交易计划”，不是确定性预测。需要给交易建议时，必须同时给出交易假设、失效条件、风险点、不交易条件和替代场景。

历史复盘的目标是让用户学会价格行为，而不是给过去走势贴名字。允许分析和标注前面走势，但每个历史复盘标签都必须还原“当时背景 -> 当时能看到的证据 -> 对应的价格行为逻辑 -> 当时可执行的动作/不动作 -> 当时的失效点 -> 后续如何验证或推翻”。最后一段行情必须单独给出下一步交易计划。

## 工作流

1. 判断请求类型：
   - 标注分析图：用户给出 K 线截图，希望在同一张图里看到教学型历史复盘和接下来交易计划。
   - 交易计划图：围绕当前最后一段走势，先建立交易假设，再标出触发、失败条件、止损和目标区域；如果图上有明显历史结构，也保留必要复盘证据。
   - 交易复盘图：在原图上标出当时背景、形态质量、进出场问题和更优处理。
   - 概念解释：解释价格行为术语、阿布体系、某个形态或市场周期；没有图时只输出文字和示意说明。
   - 学习训练：把一段行情拆成读图练习、问题清单或复盘模板。

2. 检查输入是否足够：
   - 标注分析图必须有图表截图或可渲染的 OHLC 图表数据。
   - 交易建议至少需要最后可见价格区域、周期、关键结构或用户提供的交易意图，并且必须能定义交易假设、失败条件和目标磁体。
   - 复盘至少需要进场、出场、止损或预期，以及当时图表背景。
   - 信息不足但可继续时，输出“条件性标注”；信息不足会导致编造时，先只问最少的关键问题。

3. 按固定顺序读图：
   - 分层：先把图分成“历史复盘区”和“当前计划区”。历史复盘区解释已经发生的关键交易机会；当前计划区回答下一步怎么做。
   - 均衡状态：判断价格是在交易区间中部、区间边界、突破测试，还是已经脱离旧均衡并寻找新均衡；区间中部默认降级为观察，区间边界优先准备失败突破和反向场景。
   - 市场循环：判断当前更接近急速、窄幅通道、宽幅通道、交易区间，还是状态不明；阶段不明时先按更保守的一侧处理。
   - 持续方向：如果必须选边，当前更偏多、偏空，还是区间中性；说明改变判断需要什么证据。
   - 当前决策点：确定最后一根可见 K 线、当前价格所在区域、离最近失败条件和目标磁体有多远。
   - 交易假设：在给任何入场建议前，先把当前计划命名为一种价格行为模型，例如失败突破、突破回调、趋势回调二次进场、区间看衰突破、急赴磁体、通道尾端反转或观察计划；不能命名且不能说明逻辑时，不给交易计划。
   - 磁体与关键位置：标出会改变行为的支撑阻力，包括重要前高前低、区间边界、趋势线/通道线、EMA20、突破点、测量目标和明显磁吸位；磁体只给目的地，不替代触发。
   - 等距与 50%：如果使用目标位，说明它来自 AB=CD、急速高度、形态/区间高度，还是前一波段 50% 回撤；只把它当成反应区，不当成必到点。
   - 运动分段：用阴阳交替、回调 K、显著高低点和规模匹配来切分趋势、波段和推动；二推或三推后优先警惕止盈、回调或小型反转。
   - 逐根信息：读取最近一段每根 K 线的实体、影线、收盘位置、重叠和后续跟随；至少判断关键 K 线相对前一根是突破成功、突破失败、停顿还是反转。
   - 多空力量：连续强 K、实体大小、影线、重叠程度、回调深度、突破后的后续跟随。
   - 形态质量：信号 K、入场 K、跟进 K 是否配合；信号是否顺着交易方向、是否在支撑阻力或区间边界附近、是否处在趋势尾端或区间中央。
   - 建仓方案：如果给出具体入场，说明它属于急速收线追进、首次逆势失败后顺势恢复、双重顶底旗形、楔形旗形、区间看衰突破、急赴磁体、区间突破回调、左侧通道线反转、右侧突破回调反转、20 均线缺口或第一均线缺口；每个方案都必须有触发、失败条件、止损、目标和删除条件。
   - 突破与测试：如果刚突破，标出突破点、回调测试和测试时的多空反应；没有跟进或测试失败时，不把突破当成已经成立。
   - 回调状态：判断回调是在健康测试、过深、过久，还是已经变成区间；过深或过久时，先降低顺势预期。
   - 反转与高潮：遇到反转主张时，检查反向压力、趋势线突破、结构破坏和原趋势极值测试；遇到高潮时先防耗尽和止盈，不直接逆势摸顶摸底。
   - 区间边界：如果进入双边交易，标出上沿、下沿、中轴和窄幅区间；窄幅区间内默认不交易，等待强突破和测试。
   - 交易数学与风格：估计概率、风险、潜在回报和失效点；说明计划更像波段还是剥头皮，并让盈亏比、持仓预期、市场脉络和离场管理一致。
   - 历史复盘链：每个关键历史点都压成 `当时背景 -> 当时证据 -> 价格行为逻辑 -> 当时计划 -> 当时失效 -> 后续验证`。
   - 当前交易链：把下一步结论压成 `当前判断 -> 交易假设 -> 触发证据 -> 执行动作 -> 失败条件/止损 -> 目标磁体 -> 删除条件`；如果同一位置出现多套方案，先判断是否方向冲突或重复承担同一风险，不能清楚分支时降级为观察。

4. 设计标注图：
   - 在同一张图上同时放两类标注：历史复盘标注和当前计划标注。
   - 历史复盘标注只选关键转折和关键交易点，不从左到右贴满所有波段。每个标签必须包含当时背景、证据和价格行为逻辑，不只写事后名称。
   - 当前计划标注必须放在最后可见价格附近，标出当前判断、交易假设、触发位、失败/止损位、目标区、当前不交易理由。
   - 视觉权重建议：历史复盘 50%-60%，当前计划 40%-50%。如果用户强调“接下来怎么做”，当前计划权重上调。
   - 使用半透明色块表示交易区间、窄幅震荡、通道或供需区域；使用箭头表示方向切换、突破失败、买卖信号；使用水平线表示关键价位和测量目标。
   - 历史复盘标签优先写“当时背景 + 当时证据 + 行为逻辑”，例如 `空头趋势反弹弱：只等二次做空`、`区间上沿失败：多头止盈，空头找信号`。
   - 当前计划标签优先写“交易假设 + 触发/失败/目标”，例如 `假设：下沿失败跌破 -> 目标中轴`、`突破回测多：跌回区间则删除`。
   - 图上文字必须放在可读底板上，不能超出画布，不能被半透明色块、K 线或价格轴挡住；右侧和底部标签要预留边距。
   - 不把大段解释塞进图里；详细逻辑放在图后面的交易建议里。

5. 生成标注产物：
   - 先根据读图结果写出 annotation JSON，格式见 `references/annotation-spec.md`。
   - 使用 `scripts/render_annotations.py` 把原图和 annotation JSON 渲染为标注图；输出后缀为 `.png` 时生成位图，输出后缀为 `.svg` 时生成自包含 SVG。
   - 默认优先交付 PNG，必要时同时交付 SVG 和 annotation JSON，方便继续编辑。

6. 输出结构按任务选择：
   - 标注分析图：`分析图路径 -> 历史复盘要点 -> 当前判断 -> 交易假设 -> 证据链 -> 触发条件 -> 失败/止损 -> 目标区域 -> 删除条件 -> 不交易条件`。
   - 交易计划图：`分析图路径 -> 当前是否可交易 -> 当前判断 -> 交易假设 -> 入场触发 -> 失败/止损 -> 目标区域 -> 删除条件 -> 不交易条件`。
   - 交易复盘图：`分析图路径 -> 当时背景 -> 当时证据 -> 价格行为逻辑 -> 当时计划 -> 失败点 -> 后续验证 -> 可复用规则`。
   - 概念解释：`定义 -> 在图上的识别方式 -> 有效背景 -> 失败信号 -> 常见误用`。

7. 交付前自检：
   - 已经产出可打开的标注图文件，或明确说明为什么当前不能生成图。
   - 没有用孤立形态替代背景判断。
   - 没有把低确定性的场景写成结论。
   - 没有遗漏交易假设、失败场景、止损逻辑或不交易条件。
   - 止损和目标都能从交易假设推出，不是先猜方向再补位置。
   - 历史复盘没有停留在事后命名；每个历史点都说明当时背景、当时证据、价格行为逻辑、当时计划、失效点和后续验证。
   - 当前计划没有缺席；图和正文都明确最后可见价格附近的交易假设、触发、失败/止损、目标和删除条件。
   - 图上文字不遮挡最关键的 K 线、价格轴或当前最后一段走势；文字不超出图片边界，且没有被色块压住。
   - 如果使用了实时事实，已经说明数据来源和时间。

## 资源

需要调用阿布价格行为核心模型、术语、读图顺序和常见误用时，读取 `references/price-action-framework.md`。

制作标注图前，读取 `references/annotation-spec.md`，按其中 JSON 结构规划标注元素。

生成分析图时，执行 `scripts/render_annotations.py <chart-image> <annotations.json> <output.png|output.svg>`。该脚本只负责确定性渲染，不替代价格行为判断。

最终输出前，尤其是图表分析、交易计划和复盘任务，读取 `references/quality-gate.md` 做质量检查。
