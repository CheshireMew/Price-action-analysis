# 价格行为分析

## 核心原则

这个 skill 的主产物是一张专业价格行为训练标注图，而不是单纯文字解读。用户给出 K 线图时，必须尽量在原图上叠加价格行为标注，并附上从图表证据推导出来的交易计划。

默认产物按“训练图”标准做，不按“快速看盘提示图”标准做。除非用户明确只要简洁计划，否则图上必须能看出市场周期如何切换、持续方向如何改变、关键 K 线为什么改变交易计划，以及下一步等待哪类信号 K、入场 K 和跟进 K。

标注阶段默认采用克制模式。先判断这张图最需要教会用户什么，再按重要性只保留少量标注。优先使用区间框、通道边界线、关键水平线、虚线、简短说明和右侧计划面板；默认不使用指向线、编号或圈号。只有用户明确要求逐根编号，或原图本来就有 bar count/编号时，才在图上新增编号；图面看不清时，删除低优先级标注，把解释移到正文。

克制只约束图面密度，不约束正文解释深度。触发条件：图上删除了低优先级文字、只保留少量对象、或用户要求学习/训练/详细解释。先保留图面清楚，再在正文补齐每个上图对象的完整解释。通过标准：正文能逐项说明每个框、线、通道、失败突破和当前计划分别解释哪段走势、证据是什么、交易动作怎么变。失败处理：如果正文也无法讲清这个对象，就从图上删除；如果对象重要但图面放不下，就保留框线，把解释移到正文。

图上每一个标注都必须先绑定到一个明确对象。对象可以是一个区间框、一组通道边界、一条水平线、一次失败突破、一段当前回调或最后决策区域。写文字前先说清：这个文字解释哪一个对象、对象边界在哪里、它提供什么价格行为证据、观众要学到什么动作。说不清就不能把文字框放到图上。

标注对象不是凭空挑出来的，必须先来自“走势分段草稿”。先从左到右把图切成 5-7 段，给每段写清角色：强趋势、横盘、突破、突破失败、回调通道、反弹、当前小区间。再判断每段是谁控制市场、控制权在哪里改变、这一段是否会改变交易动作。只有状态切换、控制权切换、关键边界和当前决策段，才进入图上标注。

用户圈出、框出、箭头指出、截图裁剪放大，或在文字里点名某一段走势时，先把该区域作为候选一级对象处理。处理顺序：先定位这个区域的起点、终点和上下边界；再判断它是状态切换段、控制权切换段、当前决策段、还是只说明背景；再决定上图或放正文。通过标准：最终输出必须明确解释这段走势为什么重要或为什么不重要。失败处理：如果它能解释当前计划、持续方向或状态切换，必须上图或在正文单独成段；只有它不改变任何交易动作时，才降级为背景说明，并说清降级理由。

图上文字不是贴结论。每个历史说明必须贴近它解释的对象，并写成“看到什么 -> 说明什么 -> 怎么做”；每个当前计划必须贴近最后价格或放在右侧计划面板，并写成“当前位置 -> 需要什么证据 -> 满足后做什么 -> 不满足怎么处理”。如果文字离对象远、对象边界不清或观众需要猜它讲的是哪里，标注失败。

当前计划面板的位置服从可读性和对象绑定，不固定在右侧。触发条件：最后价格右侧空间不足、用户要求放到中央下方、或计划需要更完整地说明多空触发和删除条件。先找不遮挡 K 线、价格轴和关键结构的空白区；再把计划面板放在最后决策对象附近、右侧、中央下方或单独第二张计划图；最后用图后正文补齐完整交易链。通过标准：面板能读出当前是否可交易、主假设、触发、失败/止损、目标和不交易条件。失败处理：如果一张图放不下，就拆成“结构复盘图”和“当前计划图”。

把用户提供的图表、OHLC 数据或明确行情片段作为唯一事实源。不要凭空补图、补价格、补成交细节；如果用户问当前行情、最新价格或具体标的的实时判断，先获取最新数据或要求用户提供图表，再进入分析。

使用阿布价格行为学作为主要分析语言：先判断背景和市场周期，再看具体形态。图上主标签必须优先使用价格行为学经典中文术语，例如买入高潮、卖出高潮、突破失败、强突破、突破回测、两段下跌、测试前高、交易区间、回调通道、持续多头、持续空头、测量运动。除非用户要求保留英文，英文术语只放在正文补充，不放进主标注。遇到 Always In、TTR、Bear Surprise、Sell Climax、Measured Move 等阿布术语时，图上用“持续多头/持续空头、窄幅区间、强空头意外、卖压高潮、测量运动”等中文术语，正文可括号补英文。

术语必须用“双层表达”。第一层保留经典术语，第二层用普通话解释新手该看什么，第三层写交易动作或风险。例如 `买入高潮 -> 突破失败 -> 先防两段下跌/大回撤`，不要只写“不追高”；例如 `交易区间 -> 上下突破都没跟进 -> 中部少做`，不要只写“震荡”。如果术语放不下解释，就把术语留在图上，解释放正文；不能为了通俗而删掉专业术语，也不能只贴术语让新手猜。

价格行为分析不是“上破做多、下破做空”的条件罗列。每个可交易计划都必须先说清交易假设，例如“区间下沿失败跌破后回到中轴”“空头趋势中的弱反弹二次做空”“强突破后的回测多头”“通道尾端反转尝试”。如果只给价格线，没有信号 K、入场 K、跟进 K 和失败后怎么处理，就降级为观察，不给入场计划。

交易计划必须按同一条逻辑链推导：市场周期 -> 当前价格位置 -> 交易假设 -> 入场触发 -> 失败条件 -> 止损位置 -> 目标磁体 -> 删除条件。止损来自“这个判断在哪里被证明错”，目标来自前高前低、区间中轴/边界、突破点、测量运动、AB=CD、趋势极值或其它图上磁体；不能先给方向再随手补止损和目标。

读图只抓两件事：脉络和动能。脉络回答市场现在处于趋势、通道、区间、突破还是测试；动能回答多空谁在推进、推进是否连续、是否有跟进。任何形态名称都必须服从这两项判断。

默认把市场理解为“动态寻找公平价”的过程。读图时先判断价格是在均衡区间内来回测试，还是已经进入寻找新均衡的失衡运动；突破先按边界测试处理，只有出现强突破和后续跟随，才升级为新一段趋势或新交易区间的探索。

把每一根 K 线都视为有交易理由的成交结果。逐根读取信息，先按波段和状态分段，再只把会改变交易计划的结构画到图上；逐根依据写进正文。不要把看不懂的 K 线称为噪音，也不要用“庄家操纵”“割韭菜”替代可验证的多空证据。

图表事实优先于新闻、基本面叙事和多周期暗示。用户同时给出这些信息时，先回到当前图表、当前周期和最近 80-100 根 K 线；除非用户明确要求多周期分析，否则不要让更大周期或新闻理由覆盖当前图表上的止损和失效条件。

输出目标是“一张图同时完成教学型历史复盘和下一步交易计划”，不是确定性预测。需要给交易建议时，必须同时给出交易假设、失效条件、风险点、不交易条件和替代场景。

历史复盘的目标是让用户学会价格行为，而不是给过去走势贴名字。允许分析和标注前面走势，但每个历史复盘标签都必须还原“当时背景 -> 当时能看到的证据 -> 对应的价格行为逻辑 -> 当时可执行的动作/不动作 -> 当时的失效点 -> 后续如何验证或推翻”。最后一段行情必须单独给出下一步交易计划。

复盘训练必须模拟实盘，不允许只站在完整图的上帝视角命名走势。看到每个关键 K 或结构切换时，都要问：当时还不知道后面会怎么走，是否应该做决定；如果入场，信号 K、入场 K、止损、目标和删除条件在哪里；如果不入场，是因为位置差、跟进不足、空间不够，还是交易前提不成立。

专业训练图要同时回答三层问题：第一层是市场状态，说明这里是急速、窄幅通道、宽幅通道、交易区间、突破测试、反转尝试还是高潮；第二层是逐根证据，标出关键 bar、推数、信号 K、入场 K、跟进 K、强突破、失败突破和测试；第三层是执行边界，说明当时/当前应该做、等、离场、缩小目标还是删除计划。

## 工作流

1. 判断请求类型：
   - 专业训练标注图：用户要求学习、训练、专业价格行为标准、阿布式复盘，或拿别人高密度价格行为图对比；默认按逐根训练图交付。
   - 标注分析图：用户给出 K 线截图，希望在同一张图里看到教学型历史复盘和接下来交易计划。
   - 交易计划图：围绕当前最后一段走势，先建立交易假设，再标出触发、失败条件、止损和目标区域；如果图上有明显历史结构，也保留必要复盘证据。
   - 交易复盘图：在原图上标出当时背景、形态质量、进出场问题和更优处理。
   - 复盘训练图：用户要求通过复盘提高实盘能力、训练读图、找交易机会、模拟当时决策；默认按“市场周期 -> 切换信号 -> 结构/形态 -> 关键 K -> 实盘模拟问题”交付。
   - 概念解释：解释价格行为术语、阿布体系、某个形态或市场周期；没有图时只输出文字和示意说明。
   - 学习训练：把一段行情拆成读图练习、问题清单或复盘模板。

2. 检查输入是否足够：
   - 标注分析图必须有图表截图或可渲染的 OHLC 图表数据。
   - 交易建议至少需要最后可见价格区域、周期、关键结构或用户提供的交易意图，并且必须能定义交易假设、失败条件和目标磁体。
   - 复盘至少需要进场、出场、止损或预期，以及当时图表背景。
   - 信息不足但可继续时，输出“条件性标注”；信息不足会导致编造时，先只问最少的关键问题。

3. 按固定顺序读图：
   - 分层：先把图分成“逐根训练区”和“当前计划区”。逐根训练区解释市场状态如何一段段切换；当前计划区回答最后一段怎么等信号、怎么执行、怎么删除。
   - 走势分段草稿：从左到右切出主要段落，每段只写一句 `这段是什么 -> 谁在控制 -> 为什么 -> 交易动作怎么变`。例如 `上涨后横盘 -> 多头不再轻松推进 -> K线重叠且上下无跟进 -> 区间中部少做`。分段草稿是内部步骤，不必原样输出，但后面的图上标注必须能追溯到这些段落。
   - 用户强调区域处理：如果用户圈出、框出、箭头指出、放大截图或文字点名某段走势，把它插入分段草稿，优先判断它是否解释当前计划。该区域不能被当作普通背景直接略过；如果降级，必须在正文说明它没有改变交易动作的原因。
   - 段落角色筛选：给每段分配角色，包括背景段、状态切换段、控制权切换段、当前决策段和可移到正文的说明段。背景段默认少画；状态切换和控制权切换优先画；当前决策段必须画；只是在复述历史涨跌的段落移到正文。
   - 关键点处理：先识别会改变市场状态或计划的 K 线，例如强突破、第一次回调、二次进场、高潮、失败突破、区间边界测试和最后决策点。原图没有 bar 编号时，不在图上新增编号；优先用框、线和贴近结构的文字说明。只有原图已有编号，或用户明确要求逐根编号时，才少量补编号。
   - 状态切换线：按时间顺序写出 `急速/通道/区间/突破测试/反转尝试/高潮` 的切换点。每次切换都必须有 K 线证据，例如连续实体、回调变深、重叠增加、强反向突破、突破无跟进。
   - 均衡状态：判断价格是在交易区间中部、区间边界、突破测试，还是已经脱离旧均衡并寻找新均衡；区间中部默认降级为观察，区间边界优先准备失败突破和反向场景。
   - 市场循环：判断当前更接近急速、窄幅通道、宽幅通道、交易区间，还是状态不明；阶段不明时先按更保守的一侧处理。
   - 周期切换信号：每次从区间变趋势、趋势变通道、通道变区间、突破变失败突破时，必须指出当时已经能看到的信号，例如连续两三根同向强 K、突破后跟进、K 线重叠突然增加、逆势方开始轻松赚钱、强反向 K 收在极端位置。
   - 持续方向：如果必须选边，当前更偏多、偏空，还是区间中性；在图上标出持续方向改变的位置，并说明改变判断需要什么证据。
   - 当前决策点：确定最后一根可见 K 线、当前价格所在区域、离最近失败条件和目标磁体有多远。
   - 交易假设：在给任何入场建议前，先把当前计划命名为一种价格行为模型，例如失败突破、突破回调、趋势回调二次进场、区间看衰突破、急赴磁体、通道尾端反转或观察计划；不能命名且不能说明逻辑时，不给交易计划。
   - 磁体与关键位置：标出会改变行为的支撑阻力，包括重要前高前低、区间边界、趋势线/通道线、EMA20、突破点、测量目标和明显磁吸位；磁体只给目的地，不替代触发。
   - 等距与 50%：如果使用目标位，说明它来自 AB=CD、急速高度、形态/区间高度，还是前一波段 50% 回撤；只把它当成反应区，不当成必到点。
   - 运动分段：用阴阳交替、回调 K、显著高低点和规模匹配来切分趋势、波段和推动；二推或三推后优先警惕止盈、回调或小型反转。使用双顶、楔形、抛物线或末端旗形时，必须把第一推、第二推、第三推标出来。
   - 逐根信息：读取最近一段每根 K 线的实体、影线、收盘位置、重叠和后续跟随；至少判断关键 K 线相对前一根是突破成功、突破失败、停顿还是反转。
   - 多空力量：连续强 K、实体大小、影线、重叠程度、回调深度、突破后的后续跟随。
   - 形态质量：信号 K、入场 K、跟进 K 是否配合；信号是否顺着交易方向、是否在支撑阻力或区间边界附近、是否处在趋势尾端或区间中央。
   - 建仓方案：如果给出具体入场，说明它属于急速收线追进、首次逆势失败后顺势恢复、双重顶底旗形、楔形旗形、区间看衰突破、急赴磁体、区间突破回调、左侧通道线反转、右侧突破回调反转、20 均线缺口或第一均线缺口；每个方案都必须有触发、失败条件、止损、目标和删除条件。
   - 突破与测试：如果刚突破，标出突破点、回调测试和测试时的多空反应；没有跟进或测试失败时，不把突破当成已经成立。任何“突破做多/跌破做空”必须改写成“突破 K -> 跟进 K -> 回测不破/失败突破 -> 执行动作”。
   - 回调状态：判断回调是在健康测试、过深、过久，还是已经变成区间；过深或过久时，先降低顺势预期。
   - 反转与高潮：遇到反转主张时，检查反向压力、趋势线突破、结构破坏和原趋势极值测试；遇到高潮时先防耗尽和止盈，不直接逆势摸顶摸底。
   - 区间边界：如果进入双边交易，标出上沿、下沿、中轴和窄幅区间；窄幅区间内默认不交易，等待强突破和测试。
   - 交易数学与风格：估计概率、风险、潜在回报和失效点；说明计划更像波段还是剥头皮，并让盈亏比、持仓预期、市场脉络和离场管理一致。
   - 历史复盘链：每个关键历史点都压成 `当时背景 -> 当时证据 -> 价格行为逻辑 -> 当时计划 -> 当时失效 -> 后续验证`。
   - 实盘模拟链：每个可交易点都压成 `当时看到什么 -> 是否允许入场 -> 如果入场，止损在哪里 -> 目标在哪里 -> 哪根 K 出现后前提失效 -> 是否需要提前离场或等待下一次机会`。不能回答止损、目标和失效时，标为训练观察点，不标为交易机会。
   - 当前交易链：把下一步结论压成 `当前判断 -> 交易假设 -> 触发证据 -> 执行动作 -> 失败条件/止损 -> 目标磁体 -> 删除条件`；如果同一位置出现多套方案，先判断是否方向冲突或重复承担同一风险，不能清楚分支时降级为观察。

4. 设计标注图：
   - 先把走势分段草稿压缩成标注故事线：`背景如何开始 -> 哪里进入区间/通道 -> 哪根 K 或哪段走势改变控制权 -> 当前价格处在哪个决策区`。故事线说不清时，先重读图，不直接画标注。
   - 再做标注对象绑定表，内部写清 `图上对象 -> 来自哪一段走势 -> 这段角色 -> 价格行为证据 -> 观众要学什么 -> 图上怎么标 -> 文字放哪里`。绑定表填不清的对象，不进入图上标注。
   - 对用户强调区域单独做绑定：写清 `用户强调区域 -> 边界 -> 状态角色 -> 为什么影响/不影响当前计划 -> 上图方式或正文解释位置`。这条绑定表没有填完前，不能交付。
   - 在同一张图上只放三类标注：结构对象、当前决策对象、少量必要证据对象。默认最多 3-5 个对象；超过后先拆成两张图，或把低优先级对象放到正文。
   - 先按优先级删选标注：一级必须保留市场状态/区间框、关键支撑阻力/突破点、当前计划面板；二级按空间选择通道边界线、测量线、1-3 个关键说明；三级放到正文，包括逐根复盘、完整术语解释和多套备选计划。
   - 先选标注策略：趋势/通道优先用两条清楚边界线；区间优先用矩形框、上下沿和中轴；突破优先用突破点、跟进 K、回测线；测量目标优先用括号线、虚线和锚点；复杂教学优先写在正文，不硬塞进图里。
   - 逐根训练标注默认不铺满图面。图上只画会改变市场状态或当前计划的证据；关键 bar 复盘、推数、信号 K/入场 K/跟进 K 的完整解释放到正文。
   - 历史复盘标注只选关键转折和关键交易点，不从左到右贴满所有波段。每个标签必须包含当时背景、证据和价格行为逻辑，不只写事后名称。
   - 当前计划标注必须和最后决策对象绑定，标出当前判断、交易假设、触发位、失败/止损位、目标区、当前不交易理由。默认贴近最后价格；如果右侧空间不足、用户要求中央下方，或计划需要更完整说明，就放到中央下方空白区或拆成第二张计划图。
   - 视觉权重建议：专业训练图中逐根训练和历史复盘 60%-70%，当前计划 30%-40%；用户明确只问“接下来怎么做”时，当前计划权重上调。
   - 使用统一视觉语法：区间只能用矩形框；通道只能用两条清楚边界线；关键位只能用水平线；计划线用虚线；不画无法命名用途的斜线。
   - 文字框应尽量贴近它解释的结构。默认不用指向线、圆圈、椭圆或带数字的 marker；如果文字离对象太远，先移动文字或删减文字。只有能明确说明“这个文字指向哪个具体对象”时，才使用短、细、灰色的指向线。
   - 文字框必须成为某个对象的标签。区间文字放在区间框内或紧贴框边；通道文字放在通道边界旁；突破失败文字放在失败突破那段附近；当前计划放在最后价格附近、右侧计划面板、中央下方空白区或第二张计划图。不能把文字框散落在空白处。
   - 图例只在颜色或线型不直观时添加，且不超过两行。若图例会增加拥挤，改在正文解释颜色含义。
   - 如果图面信息密度高，可以输出两张图：第一张做专业训练复盘，第二张只保留当前计划。不要为了塞进一张图而把文字压到看不懂。
   - 历史复盘标签优先写“当时背景 + 当时证据 + 行为逻辑”，例如 `空头趋势反弹弱：只等二次做空`、`区间上沿失败：多头止盈，空头找信号`。
   - 教学型分段标签优先写“段落角色 + 控制权变化 + 动作”，例如 `大涨后横盘：上下都缺跟进，中部少做`、`大阴线打回：突破失败，转看回调/空头通道`、`创新高无跟进：多头降级，先看弱反弹`。
   - 图上标签优先采用三行结构：第一行经典术语，第二行人话解释，第三行动作含义。示例：`买入高潮 / 突破失败 / 先防大跌`、`两段下跌 / 空头已有跟进 / 反弹先当测试`、`突破回测 / 回测守住 / 再看多`。如果只能写一句，必须同时包含术语和动作，例如 `买入高潮失败：先防大回撤`。
   - 当前计划标签优先写“交易假设 + 触发/失败/目标”，例如 `假设：下沿失败跌破 -> 目标中轴`、`突破回测多：跌回区间则删除`。
   - 图上文字必须放在可读底板上，不能超出画布，不能被半透明色块、K 线或价格轴挡住；右侧和底部标签要预留边距。
   - 每个色块和线条都必须一眼能解释用途。区间用矩形框，通道用两条清楚边界线；说不清用途的斜线、扭曲色块、装饰性箭头直接删除。
   - 不把大段解释塞进图里；详细逻辑放在图后面的交易建议里。

5. 生成标注产物：
   - 先根据读图结果写出 annotation JSON，格式见 `references/annotation-spec.md`。
   - 使用 `scripts/render_annotations.py` 把原图和 annotation JSON 渲染为标注图；输出后缀为 `.png` 时生成位图，输出后缀为 `.svg` 时生成自包含 SVG。
   - 默认优先交付 PNG，必要时同时交付 SVG 和 annotation JSON，方便继续编辑。

6. 输出结构按任务选择：
   - 专业训练标注图：`分析图路径 -> 图上对象逐项解释 -> 市场状态切换 -> 关键 bar 复盘 -> Always In/持续方向 -> 推数与测量 -> 当前判断 -> 交易假设 -> 信号 K/入场 K/跟进 K -> 失败/止损 -> 目标区域 -> 删除条件 -> 不交易条件`。
   - 复盘训练图：`分析图路径 -> 图上对象逐项解释 -> 市场周期切换 -> 周期切换信号 -> 可交易点/观察点 -> 实盘模拟问题 -> 信号 K/入场 K/跟进 K -> 止损/目标 -> 前提失效 -> 提前离场条件 -> 下次可复用规则`。
   - 标注分析图：`分析图路径 -> 图上对象逐项解释 -> 历史复盘要点 -> 当前判断 -> 交易假设 -> 证据链 -> 触发条件 -> 失败/止损 -> 目标区域 -> 删除条件 -> 不交易条件`。
   - 交易计划图：`分析图路径 -> 当前是否可交易 -> 当前判断 -> 交易假设 -> 入场触发 -> 失败/止损 -> 目标区域 -> 删除条件 -> 不交易条件`。
   - 交易复盘图：`分析图路径 -> 当时背景 -> 当时证据 -> 价格行为逻辑 -> 当时计划 -> 失败点 -> 后续验证 -> 可复用规则`。
   - 概念解释：`定义 -> 在图上的识别方式 -> 有效背景 -> 失败信号 -> 常见误用`。

7. 交付前自检：
   - 已经产出可打开的标注图文件，或明确说明为什么当前不能生成图。
   - 没有用孤立形态替代背景判断。
   - 没有把低确定性的场景写成结论。
   - 没有遗漏交易假设、失败场景、止损逻辑或不交易条件。
   - 止损和目标都能从交易假设推出，不是先猜方向再补位置。
   - 历史复盘没有停留在事后命名；每个历史点都说明当时背景、当时证据、价格行为逻辑、当时计划、失效点和后续验证。
   - 复盘训练没有使用上帝视角；每个交易机会都回答如果当时入场，止损、目标、失效和提前离场条件在哪里。
   - 当前计划没有缺席；图和正文都明确最后可见价格附近的交易假设、触发、失败/止损、目标和删除条件。
   - 专业训练图没有缺少关键 bar、推数、持续方向、测量运动或信号/入场/跟进 K；图上只保留必要定位，正文必须补齐逐根依据。
   - 如果用户圈出、框出、箭头指出、放大或点名某段走势，已经单独解释该区域的边界、状态角色、证据、交易动作变化和是否影响当前计划。
   - 如果图上采用克制标注，正文已经逐项解释所有上图对象；没有把“图上少写”变成“正文也少讲”。
   - 当前计划面板的位置已经按可读性选择；如果放在中央下方或第二张图，仍然能看出它绑定到最后决策对象，而不是悬空结论。
   - 没有把价格位本身当作触发；所有入场触发都必须落到信号 K、入场 K、跟进 K、突破回测或失败突破证据上。
   - 图上术语都配有人话解释和动作含义；没有只有内部缩写、英文术语或孤立编号。
   - 图上关键位置已经使用价格行为学经典术语；没有把“买入高潮、突破失败、两段下跌、测试前高”等专业判断降级成“不追高、涨不动、回落”这类泛泛表达。
   - 每个图上文字框都能回答“我解释的是哪一个对象”；没有悬空结论文字框。
   - 没有在原图无编号时主动新增一串数字或圈号；没有让指向线看起来像趋势线、压力线或交易信号；关键区域能被框线本身定位。
   - 已经先做走势分段，并且图上标注能对应到这些段落；没有把每个历史波段都标成同等重要。
   - 图上文字不遮挡最关键的 K 线、价格轴或当前最后一段走势；文字不超出图片边界，且没有被色块压住。
   - 如果使用了实时事实，已经说明数据来源和时间。

## 资源

需要调用阿布价格行为核心模型、术语、读图顺序和常见误用时，读取 `references/price-action-framework.md`。

制作标注图前，读取 `references/annotation-spec.md`，按其中 JSON 结构规划标注元素。

生成分析图时，执行 `scripts/render_annotations.py <chart-image> <annotations.json> <output.png|output.svg>`。该脚本只负责确定性渲染，不替代价格行为判断。

最终输出前，尤其是图表分析、交易计划和复盘任务，读取 `references/quality-gate.md` 做质量检查。
